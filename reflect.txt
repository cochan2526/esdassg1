
This is an assignment of using Django to build a web application 
driven by database, based on a real open dataset.

Writing the 
program itself is not difficult at all as long as what to be 
developed is known, and there will be hundreds, if not thousands, 
of examples over the internet for reference.  However, finding 
the right, and permitted to use, open dataset and a suitable 
theme is not as easy as imagine and I have to admit that this 
choice of using "Fare and Barrier Free Facilities" as topic is 
found not very good during implementation, although difficulty 
is not high, it is very time consuming to work with 4 tables, 
plus one more for user preference, and several pages, and the 
filter function, which is done by user preference, may not be so 
align with the expectation of this assignment.

The dataset is used almost as is, only trimmed down and removed 
redundant and meaningless data, for the simplcity of the 
application, no normalisation is done.  Although this is alright 
for this kind of student project, it is better to normalise the 
tables if futher development or if it is expanded to be a large 
coporate scale project.
